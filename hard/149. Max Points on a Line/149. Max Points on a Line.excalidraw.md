First, we need to figure out how and where we’re going to store the points on the coordinate plane. I use a hash table for that, and to tell one pair of points from another, I rely on something called the slope. ^fEGq4J9z

The slope (also called the gradient) tells us the direction of a line on a plane. Usually it's written as m, and it's calculated by dividing the vertical change by the horizontal change — that’s the classic "rise over run" formula. ^W7Tsdv7h

Here’s the standard slope formula: ^ef5hRWlU

(y₂ - y₁) / (x₂ - x₁) or simply Δy / Δx, which is also equal to tan(θ). ^FVTXZ6y9

But using tan(θ) isn’t always the most accurate way to compare slopes — floating-point values can be tricky. So instead, I simplify the slope using the greatest common divisor (GCD). That way, each slope is stored as a pair (Δy / GCD, Δx / GCD) — a nice, clean ratio that avoids floating-point errors. ^LTq6ZAde

The hash table keeps track of how many times each slope appears. If a new point creates the same slope as one we’ve already seen, I just increase the count for that slope. ^EIBLGthJ
